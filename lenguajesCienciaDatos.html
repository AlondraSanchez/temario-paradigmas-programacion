¿Qué es la Ciencia de Datos?

La ciencia de datos es hoy en día la herramienta fundamental para la explotación de datos y la generación de conocimiento. Entre los objetivos que persigue se encuentra la búsqueda de modelos que describan patrones y comportamientos a partir de los datos con el fin de tomar decisiones o hacer predicciones.

Durante los últimos años, la cantidad de datos que circulan en la red es inmesurable. La ingente cantidad de datos disponibles, generados de forma continua e incesante por entidades, usuarios, servicios o dispositivos, ha implicado el desarrollo de nuevos métodos científicos e ingenieriles para disponer de sistemas y procedimientos capaces de almacenar, procesar y analizar dichos datos, generando de esta manera información y conocimiento en sectores tan dispares como la industria, banca, finanzas, marketing, seguros, redes sociales, comercio electrónico, salud, gobierno electrónico, ciudades inteligentes, biología, medicina o ciencias de la tierra, por citar algunos.

Big Data

Big data es un término de origen inglés cuya traducción equivale a "datos masivos". Muchas son las definiciones que entidades y organizaciones han dado para el término big data, pero todas ellas se pueden resumir en el conjunto de datos cuyo tamaño supera considerablemente la capacidad de captura, almacenado, gestión y análisis del software convencional de bases de datos.

Sin embargo, el concepto no hace referencia simplemente al tamaño de la información, sino también a la variedad del contenido y a la velocidad con la que los datos se generan, almacenan y analizan. Estas dimensiones son las "3V" con las que la empresa Gartner describió big data:

  • Volumen. Como su propio nombre indica, big data corresponde al gran volumen de datos que se generan diariamente en las empresas y organizaciones de todo el mundo.
  • velocidad. Es decir, cómo de rápido se están produciendo los datos, así como la rapidez en la que se trata de satisfacer la demanda de éstos.
  • Variedad. Esta variedad de datos incluye datos estructurados (como las bases de datos) y no estructurados, datos jerárquicos, documentos, correo electrónico, datos de medición, video, imágenes fijas, audio, transacciones financieras, etc.

Aunque, algunas organizaciones incluyen un par más de "V":

  • Veracidad. Big data debe ser capaz de tratar y analizar inteligentemente el enorme volumen de datos con el objetivo de obtener infomación verídica y &uuac til que permita realizar tomas de decisiones basada en los datos más exactos.
  • Valor. Se refiere a los beneficios que trae consigo el uso de big data, como la reducción de costes, la eficiencia operativa y las mejoras de negocio.

Por tanto, se define a la tecnología big data como el conjunto de arquitecturas y herramientas informáticas destinadas a la manipulación, gestión y análisis de grandes volúmenes de datos desde todo tipo de fuentes, diseñadas para extraer valor y beneficio de los mismos, con una amplia variedad en su naturaleza, mediante procesos que permitan capturar, descubrir y analizar información a alta velocidad y con un coste reducido.

Proceso de la ciencia de datos

El proceso de analizar y utilizar de datos sigue un flujo como el siguiente:

  • Planificación. Se debe definir el proyecto y los resultados que se esperan obtener de este.
  • Construir el modelo de datos. Los científicos de datos necesitan utilizar las herramientas y la capacidad de procesamiento adecuadas, además del acceso a los datos correctos.
  • Evaluar el modelo. Los modelos construidos deben tener un alto porcentaje de exactitud antes de que se puedan implementar productivamente. Estas evaluaciones implican métricas de evaluación y visualizaciones para medir el rendimiento del modelo frente a los datos.
  • Explicar el modelos. Los científicos de datos desean recibir explicaciones automatizadas de la ponderación relativa y la importancia de los factores que intervienen en la generación de una predicción, junto con detalles explicativos específicos del modelo sobre las predicciones del modelo.
  • Implementación del modelo. Una vez que se ha construido, evaluado y explicado correctamente el modelo, el siguiente paso es implementarlo en el sistema que lo requiere.
  • Monitorear el modelo. Los modelos creados siempre deben monitorearse después de la implementación para garantizar su correcto funcionamiento, ya que al paso del tiempo, los datos con los que se entrenó al modelo pueden ser obsoletos para futuras predicciones.

Lenguajes de programación para la ciencia de datos

Algunos de los lenguajes en los que valdría la pena especializarse como científico de datos son:

R

R es un lenguaje poderoso que sobresale en una gran variedad de aplicaciones de visualización de datos y estadística, y ser de código abierto permite tener una comunidad muy activa de contribuyentes. Su reciente crecimiento en popularidad es un testimonio de lo eficaz que es.

Python

Gran parte del proceso de ciencia de datos gira en torno a procesos ETL (extracción-transformación-carga). Esto hace que la generalidad de Python encaje perfectamente. Además, las librerías como Tensorflow de Google hacen de Python un lenguaje muy emocionante para el aprendizaje automático (machine learning).

SQL

SQL es más útil como lenguaje de procesamiento de datos que como herramienta analítica avanzada. Sin embargo, gran parte del proceso de la ciencia de la información depende de ETL, y la longevidad y la eficiencia de SQL son una prueba de que es un lenguaje muy útil para el científico de datos moderno.

Java

Muchas compañías apreciarán la capacidad de poder integrar el código de producción de ciencia de datos directamente en la base de un código ya existente, y además encontramos que el rendimiento de Java y la seguridad de tipos son muy ventajosos.

Scala

Cuando se trata de usar la computación en clúster para trabajar con Big Data, Scala + Spark son soluciones fantásticas. Si tienes experiencia con Java y otros lenguajes de tipado estático, también apreciarás las características de Scala. Sin embargo, si tu aplicación no maneja los volúmenes de datos que justifiquen la complejidad agregada de Scala, es probable que tu productividad sea mucho mayor al usar otros idiomas, como R o Python.

Julia

Julia es un lenguaje reciente, por lo que aún no cuenta con tantas herramientas como R o Python, sin embargo valdría la pena conocerlo y ver cómo evoluciona en un futuro.

MATLAB

El uso generalizado de MATLAB en una variedad de campos cuantitativos y numéricos tanto en la industria como en el mundo académico, lo convierte en una opción seria para la ciencia de datos. Un caso de uso claro sería aquel en el que una aplicación requiera una funcionalidad matemática intensiva y avanzada. De hecho, MATLAB fue diseñado específicamente para esto.
